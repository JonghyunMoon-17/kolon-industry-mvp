# 코오롱인베스트먼트 투심보고서 산업분석 MVP 설명서

## 1. 프로젝트 목적

본 프로젝트는 코오롱인베스트먼트 캡스톤 프로젝트의 일환으로, 투자심사보고서 작성 업무 중 **「산업 현황 및 분석」 섹션 초안 생성**을 자동화하는 MVP이다.

초기 목표는 투심보고서 전체 자동화였으나, 실제 내부 데이터 접근과 보안 이슈, 샘플 데이터 미매칭 문제를 고려하여 범위를 좁혔다. 현재 MVP는 IR Deck, 딜메모, IR 피칭 녹취를 입력받아 산업분석 초안을 생성하고, 추출 결과·근거·검증 결과·미니 지식그래프를 함께 보여주는 데 초점을 둔다.

핵심 메시지는 다음과 같다.

> 본 MVP는 단순 요약 챗봇이 아니라, 입력 문서에서 산업분석 관련 정보를 추출하고, 온톨로지 구조에 매핑한 뒤, 근거 기반으로 「산업 현황 및 분석」 초안을 작성하는 프로토타입이다.

## 2. 입력 데이터 구분

MVP는 입력 데이터를 두 종류로 구분한다.

### 회사 자료

특정 투자대상기업에 대한 실제 분석 근거로 사용한다.

- IR Deck: 회사 사업, 제품, 시장, 고객, 경쟁환경, 성장 논리 파악
- 딜메모: 투자검토 배경, 투자포인트, 리스크, 시장성 판단 근거 파악
- IR 피칭 녹취/메모: 창업자 설명, Q&A, 고객 문제, 경쟁 우위 보강

### 샘플 투심보고서

샘플은 정답 데이터가 아니라 **문체와 구성 참고용**이다.

- 문단 흐름
- 투자검토 보고서식 표현
- 수치와 리스크를 보수적으로 쓰는 방식
- 산업 변화와 투자대상기업 기회를 연결하는 방식

샘플 보고서의 회사명, 산업, 사실관계는 대상회사 분석에 섞지 않는다.

## 3. MVP 처리 흐름

앱의 기본 처리 흐름은 다음과 같다.

```text
문서 업로드
→ 문서 파싱
→ ParsedChunk 구조화
→ 산업분석 정보 추출
→ 미니 KG 생성
→ 근거 검색
→ 검증
→ 산업 현황 및 분석 초안 생성
```

현재 앱에는 두 가지 생성 방식이 있다.

### direct-draft

실제 초안 품질을 우선하는 방식이다.

회사 자료와 샘플 문체를 Claude에 직접 제공하여, 사람이 Claude에게 자료를 주고 “산업 현황 및 분석 파트를 작성해줘”라고 요청하는 방식에 가깝다. 현재 데모에서는 이 방식을 권장한다.

### pipeline

구조화 과정을 보여주는 방식이다.

입력 문서를 `IndustryExtraction`, `mini_kg`, `evidence_map`, `validation_report`로 나누어 보여준다. 초안 품질보다는 구조화 추출, 온톨로지 매핑, 근거 검색, 검증 흐름을 설명하는 데 적합하다.

## 4. 핵심 산출물

앱은 다음 산출물을 제공한다.

- `IndustryExtraction`: 산업, 시장, 성장요인, 고객 니즈, 경쟁환경, 기회, 리스크 등 구조화 추출 결과
- `mini_kg`: 추출 결과를 노드와 관계로 표현한 미니 지식그래프
- `evidence_map`: 각 claim과 원문 chunk의 연결
- `validation_report`: 필수 항목 누락, 근거 부족, 수치 출처 부족 여부
- 산업분석 초안: 투심보고서에 붙여 넣을 수 있는 Markdown 형식 초안

## 5. 온톨로지/KG의 역할

이 프로젝트에서 온톨로지는 VC 전체 업무 지식그래프가 아니라, 「산업 현황 및 분석」 섹션 작성에 필요한 최소 정보 구조이다.

예시 관계:

```text
투자대상기업 → 속한다 → 산업
산업 → 포함한다 → 시장
시장 → 가진다 → 성장요인
고객군 → 가진다 → 문제/니즈
산업 → 가진다 → 경쟁환경
사업기회 → 근거를 가진다 → 근거문서
리스크 → 근거를 가진다 → 근거문서
```

MVP 단계에서는 Neo4j 같은 그래프DB를 쓰지 않고, JSON 기반 미니 KG로 충분하다고 판단했다.

## 6. 사용 방법

배포 앱에서 권장 설정은 다음과 같다.

```text
LLM provider: claude-api
생성 방식: direct-draft
Claude model: auto
Anthropic API key: 비워둠
```

API 키는 Streamlit Secrets에 설정한다.

```toml
ANTHROPIC_API_KEY = "sk-ant-..."
```

업로드 방식:

```text
회사 자료:
- IR Deck
- 딜메모
- IR 피칭 녹취/메모

샘플 자료:
- 투심보고서 산업분석 샘플 1~3개 권장
```

샘플 7개를 모두 넣을 수도 있으나, 문체가 서로 다를 경우 대표 샘플 2~3개만 사용하는 편이 더 안정적이다.

## 7. 현재 범위와 제외 범위

현재 MVP에 포함되는 것:

- 문서 업로드 및 파싱
- 회사 자료 기반 산업분석 초안 생성
- 샘플 보고서 문체 참고
- 구조화 추출 결과 확인
- 미니 KG 확인
- 근거 검색 및 검증 결과 확인
- Markdown 다운로드

현재 MVP에서 제외한 것:

- 투심보고서 전체 자동화
- 실제 내부 DB 연동
- LP 보고, 포트폴리오 관리, 펀드 운영 자동화
- Neo4j/Neptune 기반 대형 지식그래프
- 샘플 7개를 정답 데이터로 학습하는 방식
- 완전한 보안/권한관리/운영 배포 체계

## 8. 발표 시 강조할 점

이 프로젝트의 포인트는 “LLM으로 보고서 하나를 대충 쓰는 것”이 아니라, 실제 VC 문서 작성 업무를 다음과 같이 쪼개어 자동화 가능성을 검증했다는 점이다.

```text
비정형 투자자료
→ 산업분석 정보 구조화
→ 온톨로지 기반 항목 매핑
→ 근거 연결
→ 검증
→ 투심보고서식 초안 생성
```

특히 보안 이슈로 실제 내부 데이터와 전체 보고서 데이터를 확보하기 어려운 상황에서, 「산업 현황 및 분석」 섹션으로 범위를 제한한 것은 단순 축소가 아니라 현실적인 MVP 범위 설정이다.

## 9. 향후 고도화 방향

- 샘플 보고서 기반 style guide 자동 추출 고도화
- 산업/기술/시장별 샘플 선택 및 가중치 적용
- PDF/PPTX 표·이미지 파싱 품질 개선
- 외부 시장자료 검색 및 출처 보강
- 수치 claim의 출처 검증 강화
- Word 문서 포맷 다운로드
- 회사 내부 보안환경에 맞춘 로컬/폐쇄망 배포 검토
